# Javascript란?

- JS란? : 웹브라우저 소프트웨어를 제어하기 위한 언어
  - 그러나 `탈웹브라우저`, 즉 웹브라우저 소프트웨어의 제어 외의 영역에서도 중요성도 대두되기 시작.
  - `웹서버`를 동작시키기 위해서도 사용 **(server side script)**  
    _ex) Node.js_
    - 웹서버와 웹브라우저를 모두 JS언어로 통일시킬 수 있는 장점이 있다.
  - `탈웹`의 영역으로도 JS가 확장된다.  
    _Ex) Google Apps Script_

**결론, JS는 단순히 웹브라우저를 제어하기 위한 영역에만 국한되지 않는다.**

- JS라는 언어를 환경(웹브라우저, node, Spread Sheet)에 맞춰 사용하고, 쓰는 명령어가 달라진다.

# Javascript 도구선택

## 기본 파일

```html
<!DOCTYPE html>		// 지금 파일의 형식(document type)
// html코드의 영역
<html>
    <head>
        <meta charset="utf-8"/>
    </head>
    <body>
        <script>	// javascript로 작성된 영역을 나타낸다.
            alert('Hello world');
        </script>
    </body>
</html>
```

**윈도우**에서는 `메모장`, **맥**에서는 `텍스트 에디트`를 이용해서 html파일을 만들 수 있다.
보다 편하게 작성하기 위해서 editor를 사용해도 좋다.  
_Ex) VSC, sublime Text(유료) 등_

<br/>

## 크롬(Chrome) 개발자도구

> Tip. 크롬(Chrome)에서 개발자환경을 쓰기 위한 방법으로는 간편하게 `F12`키를 사용해서 열 수 있다.

위에 언급한 방법을 통해 개발자 환경을 열고 `console`영역에 작성하는 것을 통해 쉽게 JS를 실행해 볼 수도 있다.

<br/>

## Editor

위에서 한번 언급한 VSC나 sublime Text와 같은 editor를 통해서 쉽게 작업을 진행할 수도 있지만 각각 마다 차이가 존재한다.

**editor에 따라 제공하는 기능도 다르고, 프로그램의 무게도 다르고, 그에 따라 본인이 익혀야되는 양 또한 달라진다**(또한 유무료 차이도 존재한다).
**_그리고 더 나은 도구를 택할수록 더욱 작업의 질을 높힐 수 있기 때문에 좋은 코드를 작성하는 것만큼 editor를 찾고 택하는 것은 중요한 일이다._**

이러한 차이가 있기 때문에 본인에게 맞는 도구를 찾아서 사용하는 것이 좋다.

> 생활코딩님 曰 " 좋은 도구를 찾고 택하는 것을 게을리해서는 안 된다 "
> BUT 도구의 효용이 좋을 수록 익히기 위한 시간이 필요한 만큼, 본인이 배우기위한 여유가 있어야한다.

# 기타 참고

## 주석(comment)

주석은 환경에선 실행되지 않고 코드에 대한 설명을 해둘 수 있는 태그이다. 방법은 `//`를 통해 한 줄 짜리 주석을 달거나 `/* */`를 통해 여러 줄의 주석을 다는 것 방법이 있다.

이 기능을 이용하면 본인이 작성한 태그에 대한 간단한 설명을 통해 본인이나 타인이 나중에 열어봤을 때 의도를 알 수 있도록 도울 수 있으며, 타인이 작성한 코드를 이해하는데도 도움이 된다.

```javascript
// 한줄짜리 주석

/* 여러줄로 작성될 수 있는 주석 22222 33333 aaaa
<div></div>
*/
```

> 짧은 주석을 통해 코드에 대한 이해를 도울 수 있다.

## 줄바꿈 & 여백

JS에서는 줄바꿈(enter)을 하면 해당 줄의 명령이 끝났다고 간주한다. 줄바꿈을 하지 않고 명령들을 구분하고자 한다면 `;(세미콜론)`을 이용해서 구분한다.

```javascript
var a=1
alert(a)

// 혹은

var a=1; alert(a);
```

그리고 가독성을 위해 여백을 주고자 할 때 `Tab키`를 이용할 수 있다.
